Solved a problem in an elegant way?

What was the problem?
In my js racer game, I wanted to be able to allow the player to choose the length of the track everytime they played, but couldn't figure out how to reset the table/racetrack length.
What problem solving techniques did you use?
I googled a lot, looked at other peoples code and tried to adapt it to my problem. I realised I could just call the page reload function after a winner was found, and in the end decided to use that.
How did you feel throughout the process? What did you learn?
I had the idea to reload the page early on but wanted to be able to code the solution manually. It was frustrating but the solution I did go with was adequate. By using reload(), it meant my other move counters, player postion etc, were reset as well so in the future I may use it if the information doesn't need to be remembered and I want to reset a program.

Explain to your non-tech friend

.map()
This allows you to copy the contents of an array, apply a function to them and put them into a new array. For instance if there is an array of numbers, numberArray, you could use .map() to add 5 to each of the elements and create an array with the new numbers. The original unchanged.

.filter()
As the name suggests this allows you to test the elements of an array to met a condition, i.e >5 and copy them, or the elements that don't meet that condition to a new array.

.reduce()
From what I understand, reduce calls a function on an arrays elements and then puts all of those elements into one element in a new array.

Recursion & Iteration

Recursion and iteration are confusingly similar. They are both how a function may be repeated but do so differently. An iteration is the same block of code executed multiple times, while recursion is function code that calls itself.
So iteration would look like;
iteration() iteration() iteration()

And recursion looks like;

recursion(recursion( recursion() ) )
